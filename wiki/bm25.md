# BM25 (Best Matching 25)

A probabilistic ranking function that scores how well a document matches a
keyword query. The default ranking algorithm in Elasticsearch and Apache Lucene.
Derived from the Probabilistic Relevance Framework developed in the
1970s–1980s (source: robertson-zaragoza-bm25-2009.md).

## Formula

```
Score(D, Q) = Σ IDF(qi) × [ f(qi, D) × (k1 + 1) ] / [ f(qi, D) + k1 × (1 - b + b × |D| / avgdl) ]
```

Where:
- `qi` — individual query terms after tokenization
- `f(qi, D)` — term frequency of qi in document D
- `|D|` — document length (number of terms)
- `avgdl` — average document length across the corpus
- `k1` — term frequency saturation parameter
- `b` — document length normalization parameter

(source: elastic-practical-bm25-part2.md)

## IDF Component

Lucene/Elasticsearch uses this IDF variant:

```
IDF(qi) = log(1 + (docCount - f(qi) + 0.5) / (f(qi) + 0.5))
```

Where `docCount` is total documents and `f(qi)` is the number of documents
containing term qi. This guarantees non-negative IDF values.
(source: elastic-practical-bm25-part2.md)

## Parameters

### k1 — Term Frequency Saturation

Controls how much repeated occurrences of a query term contribute to the score.

| k1 value | Effect |
|----------|--------|
| 0        | Only IDF matters; term frequency ignored |
| 0.01     | Score plateaus almost immediately |
| 1.2 (default) | Moderate saturation; standard choice |
| 10       | Near-linear scaling with term frequency |

(source: elastic-practical-bm25-part2.md)

### b — Document Length Normalization

Controls how much document length affects scoring.

| b value | Effect |
|---------|--------|
| 0       | Document length has no effect |
| 0.75 (default) | Moderate normalization |
| 1       | Full normalization; short documents strongly preferred |

With b > 0, a match in a short document scores higher than the same match in a
long document, because longer documents dilute term density.
(source: elastic-practical-bm25-part2.md)

## Relationship to [[tfidf]]

BM25 extends TF-IDF with two improvements:

1. **Saturation**: TF-IDF scores grow linearly with term frequency. BM25 applies
   diminishing returns — the first occurrence matters most, additional
   occurrences contribute less and less. (source: tfidf-bm25-rag-guide.md)

2. **Length normalization**: TF-IDF has no built-in length correction. BM25
   normalizes by document length relative to the corpus average.
   (source: tfidf-bm25-rag-guide.md)

## Variants

- **BM25F** — field-weighted variant that scores different document fields
  (title, abstract, body) with separate parameters
  (source: robertson-zaragoza-bm25-2009.md)
- **BM25L** — addresses a bias against long documents in standard BM25
- **BM25+** — ensures a positive lower bound on term frequency contribution

See [[bm25-python-libraries]] for implementations.

## Usage in Search Systems

BM25 is the default similarity in Elasticsearch and Lucene. Configurable
per-field via the `similarity` mapping parameter. Parameters k1, b, and
`discount_overlaps` can be tuned per index.
(source: lucene-scoring.md, lucene-search-and-ranking-blaszyk.md)
