# arxiv-digest spec

## Overview

A Claude Code slash command that searches arXiv for recent papers matching a
user's interests, ranks them by keyword relevance, and presents a curated
digest with LLM-generated summaries.

## Commands

Two commands in `.claude/commands/`:

1. **`/digest`** — runs the fetch → filter → rank → summarize pipeline.
2. **`/expand-keywords`** — reads the current topic list from the config file,
   uses Claude to suggest synonyms, morphological variants, and related terms
   (e.g. "convolution" → "convolutional", "convolved"; "DSP" →
   "digital signal processing"), presents them to the user for approval, and
   writes accepted terms back to the config.

No external API calls to an LLM provider — Claude handles summarization,
keyword expansion, and presentation directly as part of command execution.
The commands can shell out to Python scripts for arXiv fetching and filtering.

## Data source

The arXiv search API (`export.arxiv.org/api/query`). Supports keyword search
across title/abstract/category, date range filtering via `submittedDate`, and
pagination (max 2,000 results per request, 30,000 total). Requires a 3-second
delay between paginated requests.

## Configuration

All inputs live in `config.json` at the repository root.

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| topics | list of strings | Keywords/phrases of interest | `["predictive markets", "gambling", "psychology", "AI", "DSP", "Audio", "convolution"]` |
| date_range | string | Time window to search | `"past week"`, `"past month"`, or an explicit range |
| top_n | integer | Number of top-ranked articles to return | `7` |
| random_n | integer | Number of random articles to return | `3` |

## Keyword matching

Filtering and scoring use case-insensitive word-boundary regex matching
(`\b`, `re.IGNORECASE`). Each keyword in the topics list is matched as a whole
word/phrase against paper titles and abstracts. No stemming, no semantic
matching — the `/expand-keywords` command handles vocabulary coverage upfront.

### Scoring

A paper's score is the sum of keyword hits, weighted by field:
- Title match: 3 points per keyword
- Abstract match: 1 point per keyword

A keyword that appears in both title and abstract scores both. Multiple
distinct keywords matching the same paper accumulate.

## Outputs

Two mutually exclusive lists:

1. **Top ranked** — `top_n` articles ranked by keyword relevance, highest
   first.
2. **Random picks** — `random_n` articles selected uniformly at random from
   the remaining keyword-matched articles (excluding anything in the top list).

If the filtered result set contains fewer papers than `top_n`, the top list
contains all of them and the random list is empty. If the remaining papers
after top selection are fewer than `random_n`, the random list contains all
remaining papers.

Each article entry includes:
- Title
- Authors
- arXiv link
- Categories
- LLM-generated summary (2-4 sentences, generated by Claude from the abstract)

## Computation (`/digest`)

1. **Fetch** — Query the arXiv API for papers within the date range, paginating
   with a 3-second delay between requests.
2. **Filter** — Keep only papers whose title or abstract matches at least
   one keyword (case-insensitive word-boundary regex).
3. **Rank** — Score filtered papers: 3 points per title keyword hit, 1 point
   per abstract keyword hit.
4. **Select top** — Take up to `top_n` highest-ranked papers.
5. **Select random** — From the remaining filtered papers (excluding top
   selections), select up to `random_n` uniformly at random.
6. **Summarize** — Claude generates a 2-4 sentence summary for each
   selected paper based on its abstract.
7. **Present** — Display both lists inline in the chat as formatted markdown.

## Constraints

- No external LLM API calls. Claude (running the command) does all
  summarization and keyword expansion.
- Python scripts handle arXiv data fetching and keyword filtering only.
- The two output lists must be mutually exclusive — no paper appears in both.
